## Device Quirks

Some devices require special handling. These are automatically applied based on the device
configuration.

| Quirk                     | Description                               |
| ------------------------- | ----------------------------------------- |
| `TRANSACTION_CODE_1F`     | Uses 0x1F transaction ID (modern devices) |
| `TRANSACTION_CODE_3F`     | Uses 0x3F transaction ID                  |
| `TRANSACTION_CODE_9F`     | Uses 0x9F transaction ID (wireless)       |
| `EXTENDED_FX_CMDS`        | Uses extended effect command set          |
| `WIRELESS`                | Wireless device with battery support      |
| `NO_LED`                  | Device has no RGB LEDs                    |
| `LOGO_LED_BRIGHTNESS`     | Separate logo brightness control          |
| `SCROLL_WHEEL_BRIGHTNESS` | Scroll wheel LED brightness               |
| `CUSTOM_FRAME_80`         | Uses 80-byte custom frame packets         |
| `CONTROL_IFACE_3`         | Uses HID interface 3 for control          |
| `BACKLIGHT_LED_FX_ONLY`   | FX only through backlight LED             |

## Adding Device Support

To add support for a new device:

1. Identify the device's vendor ID (`0x1532`) and product ID
2. Determine the matrix dimensions (rows x columns)
3. Identify supported LEDs and effects
4. Add an entry to the appropriate YAML file in `uchroma/server/data/`

Example entry:

```yaml
- !device-config
  name: My New Keyboard
  product_id: 0x0xxx
  dimensions: [6, 22]
  supported_leds: [backlight, logo]
  quirks: [TRANSACTION_CODE_1F, EXTENDED_FX_CMDS]
```

See [Razer Protocol Reference](razer-protocol-reference.md) for protocol details.
