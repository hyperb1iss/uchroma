## Features Legend

| Feature         | Description                                  |
| --------------- | -------------------------------------------- |
| Full Chroma     | Custom frame support, all built-in effects   |
| Extended FX     | Enhanced effect command set (class 0x0F)     |
| Basic FX        | Standard effects (breathe, spectrum, static) |
| Wireless        | Battery status, wireless connectivity        |
| Fan control     | EC-based fan/power mode control              |
| Custom frame    | Per-LED addressable RGB matrix               |
| Logo brightness | Separate logo LED brightness control         |
| Profile LEDs    | Individual profile indicator LEDs            |
| Single LED      | Device has only one controllable LED         |
| ARGB header     | Addressable RGB header controller            |
| Settings only   | No RGB, but DPI/polling settings supported   |

## Device Quirks

Quirks are automatically applied based on device configuration.

| Quirk                     | Description                               |
| ------------------------- | ----------------------------------------- |
| `TRANSACTION_CODE_1F`     | Uses 0x1F transaction ID (modern devices) |
| `TRANSACTION_CODE_3F`     | Uses 0x3F transaction ID                  |
| `TRANSACTION_CODE_9F`     | Uses 0x9F transaction ID (wireless)       |
| `TRANSACTION_CODE_08`     | Uses 0x08 transaction ID (Naga X)         |
| `EXTENDED_FX_CMDS`        | Uses extended effect command set          |
| `WIRELESS`                | Wireless device with battery support      |
| `NO_LED`                  | Device has no RGB LEDs                    |
| `SINGLE_LED`              | Single LED zone only                      |
| `LOGO_LED_BRIGHTNESS`     | Separate logo brightness control          |
| `SCROLL_WHEEL_BRIGHTNESS` | Scroll wheel LED brightness               |
| `CUSTOM_FRAME_80`         | Uses 80-byte custom frame packets         |
| `CONTROL_IFACE_3`         | Uses HID interface 3 for control          |
| `BACKLIGHT_LED_FX_ONLY`   | FX only through backlight LED             |

## Adding Device Support

To add support for a new device:

1. Identify the device's vendor ID (`0x1532`) and product ID
2. Determine the matrix dimensions (rows x columns)
3. Identify supported LEDs and effects
4. Check [Razer Device Database](razer-device-database.md) for specs
5. Add an entry to the appropriate YAML file in `uchroma/server/data/`

Example entry:

```yaml
- !device-config
  name: My New Keyboard
  product_id: 0x0xxx
  dimensions: [6, 22]
  supported_leds: [backlight, logo]
  quirks: [TRANSACTION_CODE_1F, EXTENDED_FX_CMDS]
```

Run `python scripts/audit_hardware_db.py` to verify coverage against the reference database.

See [Razer Protocol Reference](razer-protocol-reference.md) for protocol details.
